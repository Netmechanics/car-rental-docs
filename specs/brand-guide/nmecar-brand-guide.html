NMECAR Brand & Theming Guide

Οδηγός εφαρμογής της οπτικής ταυτότητας στο admin dashboard, στις αναφορές και στο documentation site. Σοβαρό, λειτουργικό ύφος με το neon gradient ως υπογραφή.

01 · Foundations

Brand essence

Η NMECAR είναι μία τεχνολογική πλατφόρμα στον χώρο του automotive. Το λογότυπο συνδυάζει σταθερό σκούρο γκρι (αξιοπιστία) με ένα neon cyan→purple gradient (καινοτομία, ψηφιακή ταυτότητα). Η συνταγή του θέματος αντιγράφει αυτή ακριβώς την ισορροπία.

🎯 Επαγγελματικό

Αξιοπιστία πάνω από εντυπωσιασμό. Σκούρα κείμενα, καθαρά layouts, σταθερή ιεραρχία.

⚡ Τεχνολογικό

Το gradient είναι το χαρακτηριστικό. Χρησιμοποιείται ως υπογραφή — όχι ως φόντο.

📊 Λειτουργικό

Σχεδιασμένο για ώρες χρήσης. Αναγνωσιμότητα, contrast και ηρεμία στην οθόνη.

Ο "70-20-10" κανόνας της παλέτας

Για να μην κουράζει το νέον σε ένα εργαλείο που χρησιμοποιείται κάθε μέρα, διανέμουμε τα χρώματα ως εξής:

  • 70% Neutrals — λευκό, ανοιχτά γκρι, ink black. Backgrounds, surfaces, body text.
  • 20% Ink (#231F20) — primary buttons, navigation, headings, icons.
  • 10% Gradient/accent — CTAs, active states, KPI accents, focus indicators, brand moments.

Κανόνες

  • Πάντα Space Grotesk για headings και πάντα Inter για body — μην τα ανταλλάσσεις.
  • Mono μόνο για: ταυτότητες (VIN, IDs), τιμές με νόημα συμμετρίας, code, timestamps σε logs.

  • Σε tables, χρησιμοποίησε `font-variant-numeric: tabular-nums;` για στοίχιση αριθμών.

11 · Application

Reports

Οι αναφορές διαβάζονται και τυπώνονται. Τα χρώματα μπαίνουν με φειδώ και μόνο όπου έχουν νόημα δεδομένων.

Κανόνες για αναφορές

  • Print-safe χρώματα — μόνο ink + 1-2 accent (purple 700 ή cyan 700). Το neon δεν τυπώνεται σωστά σε ασπρόμαυρα ή χαμηλή ποιότητα.
  • Logo πάντα στο header — primary version (σκούρο σε λευκό φόντο).
  • Σήμανση brand — μία λεπτή gradient line (2-3px) κάτω από το header. Αρκετή.
  • Γραφήματα — primary series σε purple 700 (#8A1FC4), secondary σε cyan 700 (#008A8C). Tertiary σε grey 600. Όχι το νέον.
  • Tables — alternating row backgrounds με grey 50, headers με grey 100. Όχι gradient backgrounds.
  • Page numbers / footer — grey 600, font mono, 10px.

12 · Application

Documentation site

Καθαρό, μακρόστενο column για διάβασμα. Σταθερό sidebar πλοήγησης (όπως αυτή η σελίδα).

Συστατικά documentation site

  • Reading column 680-720px — βέλτιστο για παραγράφους κειμένου.
  • Sidebar 240-280px — sticky, με τρία επίπεδα ιεραρχίας.
  • Code blocks — ink background (#231F20), syntax highlighting σε cyan (keys) και purple (values). Όπως παρακάτω.
  • Inline code — grey 100 background, font mono, padding 2-6px.
  • Callout boxes — οι alerts όπως ορίστηκαν παραπάνω. Χρησιμοποίησέ τους για tips, warnings, deprecations.
  • "Edit on GitHub" / version selector — top right, ghost button.
  • Anchor links σε headings — εμφανίζονται με opacity 0 και 0.6 on hover.
  • Search bar — top of sidebar, με ⌘K shortcut hint.
  • Πλοήγηση Previous/Next — στο bottom κάθε σελίδας.

Παράδειγμα code block

// CSS variables — τα φορτώνεις μία φορά στο :root
:root {
  --nm-gradient: linear-gradient(90deg, #01FBFD, #BD41FF);
  --nm-ink: #231F20;
  --font-display: "Space Grotesk", sans-serif;
}
13 · Implementation

CSS tokens — copy & paste

Ολόκληρο το σύστημα τιμών σε CSS custom properties. Δουλεύει με κάθε framework (Tailwind, Material, Bootstrap, plain CSS, Thymeleaf templates). Πρόσθεσε τες στο global stylesheet σου.

:root {
  /* Brand gradient */
  --nm-cyan: #01FBFD;
  --nm-purple: #BD41FF;
  --nm-gradient: linear-gradient(90deg, #01FBFD 0%, #BD41FF 100%);
  --nm-gradient-diagonal: linear-gradient(135deg, #01FBFD 0%, #BD41FF 100%);

  /* Accessible brand variants (για text σε λευκό) */
  --nm-cyan-700: #008A8C;
  --nm-purple-700: #8A1FC4;
  --nm-cyan-50: #E6FEFE;
  --nm-purple-50: #F7ECFF;

  /* Neutrals */
  --nm-ink: #231F20;
  --nm-grey-900: #1A1A1F;
  --nm-grey-800: #2D2D33;
  --nm-grey-700: #4A4A52;
  --nm-grey-600: #6B6B73;
  --nm-grey-500: #8E8E96;
  --nm-grey-400: #B5B5BC;
  --nm-grey-300: #D4D4DA;
  --nm-grey-200: #E5E5E9;
  --nm-grey-100: #F1F1F4;
  --nm-grey-50:  #F8F8FA;

  /* Semantic */
  --nm-success: #10B981;
  --nm-warning: #F59E0B;
  --nm-danger:  #EF4444;
  --nm-info:    #3B82F6;

  /* Surfaces (light mode default) */
  --bg-page:    #F8F8FA;
  --bg-surface: #FFFFFF;
  --bg-muted:   #F1F1F4;
  --border:     #E5E5E9;

  --font-mono:    "JetBrains Mono", ui-monospace, monospace;

  --s-1: 4px;  --s-2: 8px;  --s-3: 12px; --s-4: 16px;

  --r-full: 999px;

14 · Reference

Do's & Don'ts

✓ Do

  • Χρησιμοποίησε το gradient ως υπογραφή — KPI top borders, primary CTAs, active states.
  • Διατήρησε λευκά/ανοιχτά φόντα για όλο το content area.
  • Σε σκούρα στοιχεία (sidebar) χρησιμοποίησε το ink #231F20, όχι μαύρο.
  • Σε reports, χρησιμοποίησε purple 700 και cyan 700 αντί για τα νέον — είναι print-safe.
  • Διατήρησε αρκετό clear space γύρω από το λογότυπο.
  • Χρησιμοποίησε mono font για IDs, ποσά και χρόνους — βελτιώνει την αναγνωσιμότητα δεδομένων.
  • Σε modes σκοτεινής εφαρμογής (π.χ. embedded screens), χρησιμοποίησε την reversed version του λογοτύπου.
  • Πάντα alt text "NMECAR" σε κάθε image of the logo.

✗ Don't

  • Μην βάζεις gradient ως page background — κουράζει και μειώνει contrast.
  • Μην χρωματίζεις body text με τα καθαρά νέον — δεν περνούν WCAG AA.
  • Μην έχεις δύο gradient buttons στην ίδια view. Μόνο μία πρωτεύουσα ενέργεια.
  • Μην παραμορφώνεις, περιστρέφεις ή αλλάζεις τα χρώματα του λογοτύπου.
  • Μην χρησιμοποιείς το λογότυπο πάνω σε φωτογραφία χωρίς overlay για contrast.
  • Μην αναμιγνύεις άλλες grandient αποχρώσεις στο UI (π.χ. πορτοκαλί-ροζ). Μόνο cyan→purple.
  • Μην προσθέτεις skeuomorphic shadows ή 3D effects — το brand είναι flat & modern.
  • Μην χρησιμοποιείς το ALLwhite logo πάνω σε ανοιχτά γκρι — δεν διαβάζεται.
