= Dashboards

Στην `1.1` το dashboard layer εκτίθεται προς τον χρήστη μόνο μέσα από ένα minimal production deliverable: το `Σύνοψη Επιχείρησης LITE`.

Στόχος αυτής της πρώτης έκδοσης δεν είναι να αντικαταστήσει τις επιμέρους οθόνες του backoffice, αλλά να δώσει γρήγορη εικόνα για:

* τι συμβαίνει σήμερα στις κρατήσεις,
* ποια είναι η πορεία εσόδων και εμπορικής απόδοσης,
* πώς αξιοποιείται ο στόλος,
* πού εμφανίζονται αποκλίσεις ή σημεία που χρειάζονται προσοχή.

== Το τρέχον dashboard της 1.1

Το dashboard που πρέπει να θεωρείται υλοποιημένο και ορατό στη `1.1` είναι:

* xref:reporting/lite-dashboard.adoc[Σύνοψη Επιχείρησης LITE]

== Dashboards που δεν θεωρούνται ακόμα εκτεθειμένα

Τα παρακάτω πρέπει να αντιμετωπίζονται στην `1.1` ως μελλοντική κατεύθυνση και όχι ως ήδη διαθέσιμο production scope:

* πλήρες executive dashboard,
* πλήρες operations board,
* αναλυτικό revenue dashboard,
* αναλυτικό fleet utilization dashboard,
* δεύτερες σελίδες ή tabs με drill-through ανάλυση.

== Σχέση με το Superset

Το `nmecar` δεν υλοποιεί μόνο του μηχανισμό σχεδίασης dashboards. Η εφαρμογή συνδέεται με dashboard που έχει δημιουργηθεί στο Superset.

Για την τεχνική σύνδεση δείτε xref:technical/dashboard.adoc[].

== Οδηγίες τεκμηρίωσης

Όταν τεκμηριώνεται νέο dashboard σε επόμενη γραμμή έκδοσης, καλό είναι να περιγράφονται:

* ο επιχειρησιακός σκοπός,
* οι βασικοί KPI δείκτες,
* τα κύρια φίλτρα,
* τα κύρια οπτικά στοιχεία,
* τυχόν προαπαιτούμενα δεδομένων ή παραμετροποίησης.
