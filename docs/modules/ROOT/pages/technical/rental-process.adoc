= Διαδικασία Ενοικίασης Οχήματος

Η ενότητα αυτή περιγράφει τη ροή ολοκλήρωσης μιας ενοικίασης, από τη στιγμή που ο πελάτης επιλέγει όχημα έως την τελική επιβεβαίωση. Καλύπτονται τόσο η ομαλή πορεία όσο και τα κρίσιμα edge cases.

== Διάγραμμα ροής

[%autowidth,frame=none,grid=none]
|===
a|image::rental_process_diagram.svg[] a|
. Ο πελάτης βλέπει τα οχήματα:
** Σαν αποτελέσματα της αναζήτησης που έχει κάνει.
** Από τη σελίδα που παρουσιάζει όλα τα οχήματα.
** Από κάποιο άλλο τμήμα με οχήματα στη σελίδα π.χ. Προσφορές

. Ο πελάτης επιλέγει το όχημα που τον ενδιαφέρει.

. Μετά την επιλογή του οχήματος:
** Ο πελάτης επιλέγει την ασφάλεια που τον ενδιαφέρει.
** Ο πελάτης επιλέγει τυχόν έξτρα εξοπλισμό που τον ενδιαφέρει.
** Ο πελάτης συμπληρώνει τα στοιχεία του.
** Ο πελάτης επιλέγει τρόπο πληρωμής.
** Ο πελάτης επιβεβαιώνει τα στοιχεία της ενοικίασης.

. Το σύστημα ελέγχει τη διαθεσιμότητα του οχήματος σε επίπεδο πινακίδας.

. Αν δεν υπάρχει διαθεσιμότητα:
** Ενημερώνεται ο πελάτης.
** Η διαδικασία ξεκινάει από την αρχή.

. Αν υπάρχει διαθεσιμότητα:
** Το σύστημα δεσμεύει την ενοικίαση.
** Το σύστημα προχωράει στον επιλεγμένο τρόπο πληρωμής.

. Αν η πληρωμή είναι αποτυχημένη:
** Ενημερώνεται ο πελάτης.
** Η διαδικασία συνεχίζει από το σημείο επιλογής τρόπου πληρωμής.

. Αν η πληρωμή είναι επιτυχημένη:
** Το σύστημα ενοικιάζει το όχημα (σε επίπεδο πινακίδας).
** Το σύστημα εκτυπώνει τα σχετικά έγγραφα.
** Ενημερώνεται/Ευχαριστείται ο πελάτης.

. Ολοκληρώνεται η διαδικασία.
|===

== Αναλυτική ροή — Sequence Diagram

Το παρακάτω διάγραμμα απεικονίζει την αναλυτική αλληλουχία κλήσεων μεταξύ πελάτη, front-end και API, συμπεριλαμβανομένων των δύο κρίσιμων edge cases: μη διαθεσιμότητας οχήματος και αποτυχίας πληρωμής.

[plantuml,rental-process-sequence,svg]
....
@startuml
skinparam defaultFontName Arial
skinparam sequenceArrowThickness 1.5
skinparam sequenceGroupBackgroundColor #EEF4FF
skinparam sequenceGroupBorderColor #4A6FA5

actor "Πελάτης" as C
participant "Front-end" as FE
participant "Booking API" as API
database "Σύστημα" as SYS

C -> FE: Επιλογή οχήματος, extras, ασφάλεια
C -> FE: Συμπλήρωση στοιχείων & τρόπου πληρωμής
C -> FE: Επιβεβαίωση κράτησης
FE -> API: POST /booking/create

group Έλεγχος διαθεσιμότητας
  API -> SYS: Αναζήτηση διαθέσιμης πινακίδας
  alt Δεν υπάρχει διαθέσιμη πινακίδα
    API --> FE: Σφάλμα — μη διαθεσιμότητα
    FE --> C: Ενημέρωση · επιλογή εκ νέου
  else Πινακίδα διαθέσιμη
    SYS -> SYS: Πινακίδα → Reserved
    SYS -> SYS: Ενοικίαση → Κρατημένο
  end
end

group Διαδικασία πληρωμής
  API --> FE: Ανακατεύθυνση στο payment gateway
  C -> FE: Εκτέλεση πληρωμής
  alt Αποτυχία πληρωμής
    FE --> C: Ενημέρωση · νέα προσπάθεια ή αλλαγή μεθόδου
    note right of C: Η ενοικίαση παραμένει\nΚρατημένο έως λήξη\nδέσμευσης (AbandonedRentalsJob)
  else Επιτυχής πληρωμή
    API -> SYS: Ενοικίαση → Μισθωμένο
    SYS -> SYS: Πινακίδα → Rented
    API --> FE: Επιβεβαίωση κράτησης
    FE --> C: Οθόνη επιβεβαίωσης + αποστολή email
  end
end
@enduml
....

NOTE: Στην περίπτωση `BANK_DEPOSIT`, η ενοικίαση παραμένει σε κατάσταση *Κρατημένο* μέχρι ο διαχειριστής να επιβεβαιώσει χειροκίνητα την κατάθεση. Δείτε xref:admin/payments.adoc[].
